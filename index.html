About

I write code that solves narrow problems well, and try to make it boringly reliable. My day-to-day is mostly Python and TypeScript, with regular detours into C++ for hardware and into Linux internals when something refuses to behave.

I gravitate toward early-stage problems where the shape of the solution isn't obvious yet. That bias pulls me toward AI tooling, payments and risk infrastructure, and embedded work where a sensor decides whether something physical happens or not.

Currently studying in the UK and using my free time to build, read papers I can mostly follow, and ship side projects faster than I can polish them.

Now

  • building An LLM agent harness for personal task workflows — turning the "assistant" pattern into something I'd actually use every day.
  • exploring Retrieval-augmented systems for small, private knowledge bases that never have to leave the device.
  • tinkering A Raspberry Pi sensor stack feeding a local dashboard — half hardware, half excuse to write more C++.
  • reading Papers on agent evaluation and a stack of books on fintech infrastructure I keep meaning to finish.
